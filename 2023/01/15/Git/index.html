Introduction

Git 是版本管控的工具，主要用途有：

  • Code hosting: Git allows you to host your code on a remote server, making it easy for multiple people to collaborate on development.
  • Version control: Git tracks every change made to files, allowing you to revert back to previous versions if necessary.
  • Branch management: Git supports branch management, allowing you to work on different branches in parallel and then merge them back into the main branch.

使用情境為多人開發同一個專案：

  • Code hosting 提供 Remote 端存放 code，便於專案的多人開發，同時Remote 端也有 Version control 與 Branch management 的功能
  • 大型專案在開發過程會有許多版本，若當前版本壞掉，需要使用先前較穩定的版本時，Version control 能方便管理
  • 每個人開發的功能不同或是一次開發多種功能時，Branch management 能避免不同功能的code在開發時互相影響

Workspace / Local Repository / Remote Repository

簡單來說，Git 的存放空間有 workspace, local repository, remote repository 三部分。

workspace 和 local repository 都在 local 端，會共用同個資料夾，差別在於 workspace 是還沒 commit 成版本的檔案，而 local repository 是有版本紀錄的檔案

Repository 是被 Git 管理的空間，local repository 存放於使用者電腦中，透過 .git 檔管理的資料夾；global repository 則是存放於雲端(GitHub)上，能使用不同電腦透過網路進行存取。

command

change files within local

  • git init
    • first time to initialize git (add .git file to the target folder)
  • git add {file_name}
    • “git add .” stand for adding all the files
  • git commit -m “comment of revise”
    • control the version of the added files with the local repository
  • git commit -am “comment of revise”
    • -am stand for add + commit

version / branch switch

  • git log
    • log the version change of the local repository
  • git reset –hard {hashcode on log}
    • reset to the version with the hashcode
  • git checkout {hashcode/branch/HEAD}
    • switch to previous version, branch, or current version (HEAD)

change files with github

  • git clone
    • first time to get the remote repository on github
  • git push
    • push the local repository to the github
  • git pull
    • pull the repository from github to local repository and workspace
  • git fetch
    • pull the repository from github to local repository
  • git merge
    • merge the local repository and workspace

to upload file to github

  1. pull (fetch+merge) files to get the latest version on github
  2. add and commit to control the version with the files we just pull down
  3. push files to github

to download file from github

  1. pull files
  2. git add and commit to control the version
